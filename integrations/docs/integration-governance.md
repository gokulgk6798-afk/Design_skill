# Integration Governance

## Purpose

Define governance expectations for AI assistant integrations.

## Review Requirements

Integration changes must be reviewed for:

- repository discovery accuracy
- context loading order
- scope control
- prompt compatibility
- template compatibility
- token usage behavior
- validation behavior
- security and confidentiality risks

## Ownership

Recommended owner: AI Systems Owner.

Supporting reviewers:

- Governance Owner
- Design System Owner
- Content Owner
- Engineering Owner

## Update Frequency

Review integrations on integration change or per release.

## Milestone Boundary

This document defines governance architecture only. It does not create an operational approval system.
