## Objective

Validate that documents are structured, concise, actionable, versioned, and reviewable.

## Checklist

- Uses a clear and descriptive document title.
- Includes purpose, overview, objectives, scope, requirements, acceptance criteria, dependencies, references, and version history where applicable.
- Uses active voice and concise language.
- Uses consistent terminology.
- Documents decisions, not assumptions.
- Includes version, status, author, reviewer, and last updated fields where applicable.
- Uses tables for structured information.
- Uses code blocks only for relevant implementation examples.
- Includes captions or titles for images and diagrams.
- Links related resources instead of duplicating content.
- Follows review and approval process.

## Pass Criteria

Pass when the document is understandable, complete, maintainable, and ready for review.

## Escalation

Escalate to Content Owner, Product Owner, Design Owner, or Technical Owner based on the unclear section.
