# Design Patterns Knowledge Architecture

## Purpose

Define the structure for future AI-readable design pattern knowledge.

## Audience

- Design System Owner
- UX Owner
- Designers
- Developers
- AI assistants

## Scope

This directory contains approved AI-readable design patterns, anti-patterns, and pattern review guidance.

## Current Modules

- `best-practices.md`
- `patterns.md`
- `anti-patterns.md`

## Required Contents Per Module

- metadata
- pattern or anti-pattern definition
- use cases
- decision criteria
- dependencies
- AI usage notes
- review criteria
- examples placeholder

- `docs/documentation/ux.md`
- `docs/documentation/components.md`
