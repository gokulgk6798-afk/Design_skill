## Best Practices

- Use the Design Kit as the source of truth.
- Prefer approved components over custom UI.
- Use approved tokens for color, typography, spacing, radius, and elevation.
- Use approved UX writing and terminology.
- Apply accessibility standards to every artifact.
- Keep layouts aligned to the approved grid and spacing system.
- Use documented patterns consistently across products and artifacts.
- Record missing details as pending rather than inventing new standards.

## Review Criteria

An artifact follows approved patterns when:

- it uses documented components or explicitly marks missing components as pending
- colors, typography, spacing, radius, and elevation come from approved tokens
- copy follows approved UX writing standards
- accessibility guidance is applied
- documentation follows approved structure and governance
- no custom design behavior bypasses an existing Design Kit rule
