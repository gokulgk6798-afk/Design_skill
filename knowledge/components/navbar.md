## Purpose

Navbars provide primary site or application navigation and access to key product areas, account actions, and grouped menus.

## Source-Backed Patterns

The source includes:

- primary navigation links
- brand/product label examples
- grouped links and menu items
- logout/account action example
- nav links with supporting text
- description-rich menu item examples
- responsive/mobile-style repeated layouts

## Anatomy

| Part | Required | Notes |
| --- | --- | --- |
| Navbar container | Yes | Holds navigation content. |
| Brand area | Optional | Source shows product/brand-like text. |
| Navigation link | Yes | Primary repeated item. |
| Dropdown/mega item | Optional | Source shows items with supporting descriptions. |
| Account/action item | Optional | Source shows logout/action examples. |

## Accessibility Requirements

Navbars must provide semantic navigation, keyboard-accessible menus, visible focus, descriptive labels, and accessible expanded/collapsed states.
