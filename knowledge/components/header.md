## Purpose

Headers provide page or product orientation, navigation access, calls to action, and high-level user/account context.

## Source-Backed Patterns

The source includes:

- product/dashboard header examples
- menu items
- messages, users, support, and help navigation examples
- new-dashboard announcement pattern
- task cards with continue actions
- tutorial/help and user identity examples

## Anatomy

| Part | Required | Notes |
| --- | --- | --- |
| Header container | Yes | Top-level orientation area. |
| Product or page identity | Optional | Source shows product/dashboard labels. |
| Navigation items | Optional | Source shows menu and support links. |
| Announcement | Optional | Source shows a dashboard preview message. |
| Action | Optional | Source shows continue/tutorial/help actions. |
| User context | Optional | Source shows user identity example. |

## Accessibility Requirements

Headers must maintain semantic navigation, clear heading hierarchy, keyboard access, and visible focus for actions.
