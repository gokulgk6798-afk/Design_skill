## Purpose

Define approved UX content guidance for clear, concise, consistent, and action-oriented communication.

## Core Standard

Effective UX writing creates a seamless experience by helping users understand, navigate, and complete tasks with confidence.

Every piece of content should:

- be clear
- be concise
- be consistent
- be action-oriented
- help users understand what is happening
- help users know what to do next

## Voice

Maintain a consistent brand voice that is:

- clear
- professional
- approachable
- human-centered

Use simple language instead of technical jargon. Focus on user goals rather than system processes. Write with clarity before creativity.

## Tone

Adapt tone based on the user's situation while keeping the same overall voice.

Use:

- encouraging language for success states
- neutral and helpful language for informational states
- calm language for warnings
- empathetic language for errors
- welcoming language for empty states

## Writing Principles

Write in sentence case. Use active voice. Keep content concise and prioritize readability.

Maintain consistent terminology across all screens. Avoid unnecessary complexity. Ensure every message helps users complete their tasks with confidence.

Every word should have a purpose, reduce cognitive effort, and contribute to a consistent, intuitive user experience.

## Related Examples and Terminology

Approved examples, terminology, button vocabulary, message patterns, capitalization rules, and date/time formats are maintained in `knowledge/ux/ux-writing.md`.

## AI Usage Rules

AI assistants must:

- use plain language
- keep content task-oriented
- avoid unnecessary jargon
- keep terminology consistent
- write in sentence case
- use active voice
- ask for missing product terminology when needed

AI assistants must not:

- create technical explanations for user-facing messages unless necessary
- expose system-level details in UI copy
- use ambiguous action labels
- add brand personality traits not listed here
