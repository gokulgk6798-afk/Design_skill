# Templates Documentation Specification

## Purpose

Define the required structure for future template documentation across presentations, documents, proposals, workflows, and reusable artifact types.

## Audience

- Content Owner
- Brand Owner
- Design System Owner
- Sales
- Marketing
- Product teams
- AI assistants

## Scope

Includes the documentation contract for template standards. Does not include actual templates.

## Contents

- template categories
- template metadata requirements
- naming rules
- versioning rules
- maintenance process
- approval process
- dependency rules
- example references

## Dependencies

- `templates/` for future templates
- `docs/documentation/presentations.md`
- `docs/documentation/documents.md`
- `docs/documentation/examples.md`
- `docs/naming-conventions.md`

## Update Frequency

On template change.

## Owner

Content Owner.
