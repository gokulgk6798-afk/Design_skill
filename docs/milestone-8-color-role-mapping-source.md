# Milestone 8 Color Role Mapping Source

This document captures the approved color role mapping provided for Milestone 8.

## Color Role Mapping

The Finstein color system is built around a semantic color architecture that ensures consistency, accessibility, and scalability across digital products. Each color is assigned a specific functional purpose rather than being used arbitrarily, enabling designers and developers to create intuitive, recognizable, and consistent user experiences.

## Brand Colors

### Primary Brand Color

Deep Blue (`#051C2C`)

The primary brand color represents trust, security, intelligence, and professionalism. It should be used for primary branding elements, major navigation components, primary headings, and high-emphasis UI elements.

Recommended usage:

- brand identity
- primary navigation
- hero sections
- primary buttons
- high-emphasis text
- footer backgrounds

### Secondary Brand Color

Bright Blue (`#03ABEC`)

The secondary brand color represents innovation, technology, and digital transformation. It is used to create visual emphasis and highlight interactive elements without competing with the primary brand color.

Recommended usage:

- secondary actions
- links
- interactive states
- selected components
- charts and data visualization
- highlights

## Semantic Color Roles

The following colors communicate system status and user feedback throughout the interface.

| Role | Color | Purpose |
| --- | --- | --- |
| Success | `#66BA20` | Successful actions, completed states, confirmations |
| Warning | `#ECAB00` | Pending actions, caution messages, warnings |
| Error | `#E93E30` | Errors, destructive actions, validation failures |
| Information | `#03ABEC` | Informational messages, tips, guidance |

## Neutral Palette

The neutral palette supports layout, readability, and interface structure while maintaining visual balance.

Usage:

- backgrounds
- cards
- dividers
- borders
- disabled elements
- secondary text
- surface hierarchy

The provided blue-gray spectrum should be used as the standard neutral scale for all interface surfaces and elevation levels.

## Heat Map Palette

The heat map palette is reserved for dashboards, analytics, reporting, and data visualization. It provides a progressive color scale that communicates increasing values while maintaining consistency with the Finstein brand identity.

Recommended usage:

- heat maps
- performance metrics
- risk indicators
- analytical dashboards
- trend visualization
- data intensity scales

This palette should not be used for standard interface components such as buttons, forms, or navigation.

## Color Usage Principles

- Assign colors based on semantic meaning rather than personal preference.
- Use the primary brand color for high-priority actions and brand recognition.
- Reserve semantic colors exclusively for communicating system status and feedback.
- Maintain sufficient contrast between foreground and background elements.
- Use neutral colors to establish clear visual hierarchy and reduce cognitive load.
- Avoid introducing new colors outside the approved brand palette without design system approval.
- Ensure consistent color application across all products, platforms, and customer touchpoints.
