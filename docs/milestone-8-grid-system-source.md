# Milestone 8 Grid System Source

This document captures the approved grid system provided for Milestone 8.

## Grid System

The Finstein Design System uses a responsive grid framework to ensure consistency, scalability, and alignment across web applications, mobile experiences, and presentation materials. A unified grid system creates predictable layouts, improves collaboration between designers and developers, and supports responsive user interfaces across multiple platforms.

## Web Grid System

The web grid is designed for enterprise dashboards, business applications, and responsive web platforms.

| Property | Value |

## Breakpoints

| Device | Width |
| --- | --- |
| Mobile | 0-767 px |
| Tablet | 768-1023 px |
| Laptop | 1024-1439 px |
| Desktop | 1440 px+ |

## Web Guidelines

- Use a 12-column grid for all desktop layouts.
- Maintain consistent gutters and margins.
- Align components to the grid whenever possible.
- Build layouts using the 8-point spacing system.
- Avoid arbitrary positioning outside the grid.

- Use a 4-column responsive grid.
- Keep primary content within safe margins.
- Maintain consistent spacing using the 8-point system.
- Design for one-handed interaction whenever possible.
- Ensure touch targets meet accessibility recommendations.

## Tablet Guidelines

- Use an 8-column grid.
- Adapt desktop layouts without simply scaling them down.
- Prioritize readability and touch accessibility.
- Allow flexible component resizing across columns.

## Presentation Guidelines

- Align all elements to the 12-column grid.
- Keep titles aligned with body content.
- Maintain generous whitespace.
- Limit text width to improve readability.
- Use consistent spacing between content sections.

## Layout Principles

### Alignment

Align components consistently using the defined column structure and spacing system.

### Consistency

Apply the same grid and spacing rules across all screens and platforms.

### Responsiveness

Allow layouts to adapt fluidly across breakpoints while preserving hierarchy and usability.

### Readability

Use appropriate content widths and whitespace to improve scanning and reduce cognitive load.

### Scalability

Build layouts that support future products, features, and responsive requirements without introducing custom grid variations.

## Grid Best Practices

- Use the defined column structure for every layout.
- Maintain consistent margins and gutters across all screens.
- Build spacing using the approved 8-point system.
- Avoid arbitrary alignment or inconsistent spacing.
- Keep content aligned to the grid to create predictable, maintainable interfaces.
- Design components to scale gracefully across mobile, tablet, desktop, and presentation formats.
- Use whitespace intentionally to establish hierarchy and improve readability.
