# Example UI Design: Employee Dashboard

## Purpose

Demonstrates how to document a UI design using approved design system standards.

## Required Sources

- `knowledge/design/`
- `knowledge/components/`
- `knowledge/ux/`
- `knowledge/patterns/`
- `tokens/source/`
- `knowledge/checklists/ui-checklist.md`
- `knowledge/checklists/accessibility-checklist.md`

## Screen Summary

Screen: Employee Dashboard

Primary goal: Help HR users monitor employee status, recent activity, and key operational metrics.

## Required Components

- Header
- Sidebar
- Cards
- Tables
- Badges
- Dropdowns
- Buttons
- Toasts

## Layout Notes

- Use the approved responsive grid.
- Use the 8-point spacing system.
- Use approved radius tokens.
- Use minimal elevation.
- Use semantic colors only for status and feedback.

## Content Notes

- Use sentence case.
- Begin actions with verbs.
- Use approved terminology.
- Keep labels short and scannable.

## Validation

Use `knowledge/checklists/ui-checklist.md` and `knowledge/checklists/accessibility-checklist.md`.
