# Example PRD: Employee Management Module

## Purpose

Demonstrates the expected structure for a product requirement document.

## Required Sources

- `knowledge/documentation-standards/documentation-guidelines.md`
- `knowledge/ux/ux-writing.md`
- `templates/documents/`
- `knowledge/checklists/documentation-checklist.md`

## PRD Structure

1. Title
2. Purpose
3. Overview
4. Objectives
5. Scope
6. Users
7. User problems
8. Requirements
9. User flows
10. Acceptance criteria
11. Dependencies
12. Analytics and success metrics
13. Accessibility considerations
14. Open questions
15. Version history

## Terminology Examples

Use approved terms such as:

- Employee
- Candidate
- Department
- Organization
- Report
- Dashboard
- Assessment

## Validation

Use `knowledge/checklists/documentation-checklist.md` and `knowledge/checklists/ux-checklist.md`.
