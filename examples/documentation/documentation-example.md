# Example Documentation: API Authentication Guide

## Purpose

Demonstrates the expected structure for technical documentation.

## Required Sources

- `knowledge/documentation-standards/documentation-guidelines.md`
- `knowledge/ux/ux-writing.md`
- `knowledge/checklists/documentation-checklist.md`

## Document Structure

1. Title
2. Purpose
3. Audience
4. Overview
5. Prerequisites
6. Authentication flow
7. Request format
8. Response format
9. Error handling
10. Security considerations
11. Examples
12. Troubleshooting
13. References
14. Version history

## Writing Rules

- Use active voice.
- Use clear headings.
- Keep examples focused.
- Avoid internal-only terms unless they are defined.
- Document recovery paths for errors.

## Validation

Use `knowledge/checklists/documentation-checklist.md`.
